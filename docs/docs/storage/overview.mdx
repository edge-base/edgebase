:::info Control Surface
- **Access Rules**: [Storage Access Rules](/docs/storage/access-rules) gate bucket reads, uploads, and deletes.
- **Hooks**: [Storage Hooks](/docs/storage/hooks) validate uploads, restrict downloads, and run file lifecycle side effects.
- **Triggers**: Storage is not a trigger-based surface.
- **Handlers**: Inline storage behavior lives under `storage.buckets[name].handlers.hooks` in `edgebase.config.ts`.
:::

## Access Rules

```typescript
// edgebase.config.ts
storage: {
  buckets: {
    photos: {
      access: {
        read() { return true },                                                // Public read
        write(auth) { return auth !== null },                                  // Auth required
        delete(auth, file) { return auth !== null && auth.id === file.uploadedBy },
      },
    },
  },
}
```
